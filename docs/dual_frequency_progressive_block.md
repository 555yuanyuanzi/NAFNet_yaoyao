# 双频渐进恢复块

## 设计动机

运动去模糊不只是细节被抹掉，还会让轮廓、结构和局部空间关系受到拖影干扰。单一路径恢复通常需要在两件事之间折中：

- 稳定地恢复低频结构
- 锐利地补回高频细节

这个模块基于一个简单但贴近任务本质的先验：

1. 先恢复粗结构
2. 再在结构引导下纠正高频细节
3. 最后不用简单拼接，而是做可变形对齐融合

这里的重点不是“为了分频而分频”，而是让恢复过程更符合运动模糊的退化方式。

## 核心故事

该模块遵循一种 **从结构到细节的渐进恢复流程**。

- **低频分支** 负责稳定粗结构，抑制大尺度拖影污染。
- **高频分支** 不是盲目增强细节，而是在低频结果的引导下做 **选择性纠偏**。
- 最后的融合采用 **低频引导的可变形对齐**，因为结构和细节分支即使分别恢复后，局部响应仍可能存在错位。

这比“普通频域增强模块”的故事更清楚：

- 低频和高频的恢复目标不同
- 高频恢复应该显式依赖低频恢复
- 最终融合应该是面向运动模糊的，而不是通用拼接

## 模块定义

设输入特征为 `x in R^{B x C x H x W}`。

### 1. 自适应低通分解

先对输入做归一化，然后提取平滑的低频成分：

- `x_n = LN(x)`
- `x_low = LowPass(x_n)`
- `x_high = x_n - x_low`

其中低通核是逐通道学习的，并通过 `softmax` 归一化，使其始终保持平滑算子的含义，而不是变成任意卷积。

### 2. 低频结构恢复

低频分支使用轻量残差卷积块，主要恢复：

- 粗轮廓
- 物体布局
- 大尺度模糊污染

输出为：

- `x_low_hat = R_low(x_low)`

### 3. 低频引导的高频纠偏

高频分支同时接收高频残差和低频恢复结果：

- `x_high_hat = R_high(x_high ; x_low_hat)`

该分支先从 `[x_high, x_low_hat]` 预测一个空间 gate，再控制高频纠偏量，避免在不确定区域过度增强细节。

### 4. 低频引导的可变形融合

最终不是直接拼接，而是先对齐再融合：

- 高频分支在 `[x_low_hat, x_base]` 的条件下进行可变形采样
- 融合 gate 由 `[x_low_hat, x_high_aligned]` 预测

最终输出为：

- `y = x + gamma * Fuse(x_n, x_low_hat, x_high_hat)`

其中 `gamma` 零初始化，以保证训练初期稳定。

## 为什么适合运动去模糊

这个设计适合运动去模糊，主要有三点：

- 运动模糊对结构和细节的破坏方式不同
- 没有结构约束的高频恢复容易产生振铃和伪纹理
- 结构与细节分支恢复后仍可能局部错位，因此可变形融合比普通相加更合理

## 在 NAFNet 中的推荐插入位置

建议顺序如下：

1. 先只插 `middle`
2. 再试 `enc4 + middle`
3. 只有前两步有效后，再考虑 decoder 侧的细化

不建议一开始就替换所有 `NAFBlock`，否则消融会很难解释。

## 与当前其他模块的关系

- `GDPM`：全局频谱提示，更偏退化感知条件调制
- `PA`：局部低频结构提取，更偏结构平滑先验
- `FBEB-v1`：较重的频域恢复块，直接参与频域特征重建

这个模块的不同点在于：

- 它是一个 **渐进恢复块**
- 它把频率分解当作恢复任务组织方式，而不是最终目标
- 它把主要恢复过程保留在空间域中

## 建议的消融设置

最小可用的消融组可以是：

1. baseline NAFNet
2. 只有低频分支
3. 低频 + 高频并行恢复
4. 低频到高频的渐进纠偏
5. 渐进纠偏 + 可变形融合

## 当前代码映射

模块代码位于：

- `basicsr/models/dfpb.py`

核心类包括：

- `AdaptiveLowPassExtractor`
- `LowFrequencyRestorer`
- `HighFrequencyRectifier`
- `LowGuidedDeformableFusion`
- `DualFrequencyProgressiveBlock`

目前这套代码已经可以通过开关接入 `NAFNet_arch.py`，默认建议先只在 `middle` 位置做验证。
